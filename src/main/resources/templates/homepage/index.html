소개

AppPaaS는 CI/CD, 자동 배포, 확장 가능 서버, 무료 SSL/도메인으로 빠른 서비스 시작이 가능하며, 이상 시 알람과 모니터링 페이지를 제공합니다.

AppPaaS는 CI/CD와 브랜치 전략에 따라 자동 배포를 구성하고, 확장 가능한 서버를 제공합니다. 무료 SSL과 도메인이 자동 설치되어 바로 서비스를 시작할 수 있습니다. 이 모든 과정은 몇 분 안에 완료됩니다. 서비스 이상 시 알람 설정과 모니터링 페이지를 제공합니다. AppPaaS를 이용하면 웹, 모바일 앱, 홈페이지, 포럼, 쇼핑몰 등을 빠르게 서비스할 수 있습니다.

1. 프레임워크 또는 DB 선택

2. 상세 정보 입력

3.서비스 빌드 및 배포

AppPaaS는 복잡한 설정없이 개발 코드만 연동하면 서버 생성부터 배포까지 자동으로 진행되는 클라우드 서비스입니다.

  • 현재위치를 불러오기 위해 SSL이 필요했습니다. EC2로 배포를 해둔 상태에서 급하게 도메인을 구입해야하는 상황이었지만, 마감기한으로 인해 다소 촉박한 상황이었습니다. 이런 상황에서 AppPaas 서버에 배포를 하였고 추가 지연 시간 없이 해결 할 수 있었습니다!

  • AppPaaS 배포 시 자동 배포 토글을 활성화 시키게 되면, 개발자가 git에서 커밋 후 push을 날리면 알아서 바로 재배포가 되더라고요.이 부분이 매우 신기했어요.

  • 서버 코드 수정하고 배포를 위해 서버를 중지한다고 프론트에 알리고, 직접 서버를 죽이고 다시 빌드해 켜는 불편한 과정이 있어서 매우 귀찮았는데 이게 클릭 하나로 이뤄지다니 백엔드 개발자에게 최고인 클라우드 서비스.

  • 모바일 버젼이 다듬어지고 활성화 되면 좋을 것 같습니다. 핸드폰으로 급하게 접속하려고 했으나, PC 버젼보다는 비교적 개선될 수 있는 부분이 있는 것 같습니다.

  • GitHub 연동 이후, 진행 단계가 상세히 나와서 직관적이고 파악하기 편했던 것 같습니다. 다만, 개발자의 입장에서는 사용이 편하긴 하나, 비개발자들 경우에는 포트 등과 같은 전문적인 용어나 섹션을 위한 별도 간단한 설명 (마우스 클릭시 키워드별 설명 팝업 창)이 있으면 좋을 것 같습니다.
